# Portfolio Decision Process

This project is a public paper portfolio for the trusted-execution AI framework.
It is a research and review artifact, not investment, tax, legal, or financial
advice.

## Core Question

If intelligence becomes cheaper and more widely available, which public companies
control the scarce layers around trusted execution?

The portfolio therefore starts from control rights rather than from AI brand
exposure. A holding must map to one or more of these layers:

- Capacity: compute, memory, foundry, packaging, power, cooling, and test.
- Cost: infrastructure that can lower the cost of useful AI work.
- Authority: systems that can authorize agents to act in enterprise workflows.
- Outcome: data, ratings, regulated workflow, or verification assets that can
  prove whether work was correct.
- Physical AI: test or robotics optionality where agentic systems meet the
  physical world.
- Risk control: cash held as an explicit option on better evidence or lower
  prices.

## Portfolio Choices

The current target allocation is intentionally split into five roles:

| Role | Weight | Reason |
| --- | ---: | --- |
| Core compounders | 57% | Microsoft, Google, and Broadcom combine enterprise distribution, AI infrastructure, authority surfaces, and custom compute exposure. |
| Bottleneck cyclicals | 18% | SK hynix, NVIDIA, and TSMC express HBM, accelerator, foundry, packaging, and AI factory constraints, but at lower size because cyclicality is higher. |
| Vertical verifiers | 8% | MSCI, Moody's, S&P Global, and Veeva test whether trusted data, ratings, regulated workflow, and domain verification become outcome-control assets. |
| Optionality | 7% | Teradyne adds semiconductor test exposure and physical-AI optionality. |
| Cash | 10% | Cash is dry powder for valuation resets, thesis changes, and missing public-market exposure. |

## Decision Rules

Each holding needs five things before it belongs in the portfolio:

1. A role in the control-rights map.
2. Source-linked evidence.
3. Clear risks.
4. A watch item that should be reviewed later.
5. A falsifier that can force reassessment.

Sizing is qualitative but disciplined. Larger weights require stronger evidence,
multiple control-right layers, durable distribution, and lower single-cycle risk.
Smaller weights are used for cyclicals, less proven monetization paths, and
portfolio experiments.

## Review Gates

Signals are prompts for manual review, not automatic trade orders.

- Capability gate: long-horizon agent progress must keep improving.
- Economics gate: risk-adjusted cost per verified task must keep declining.
- Trust gate: enterprise agents must gain governed write and execute permission.
- Valuation gate: cash is deployed only when thesis quality and price both
  justify it.

## Operating Cadence

- Prices refresh daily in the GitHub Actions workflow.
- Indicators should be reviewed monthly.
- Control-right scores should be refreshed quarterly.
- Rebalances remain manual until the framework has a longer live history.
